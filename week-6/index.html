來總結一下

你現在要開一個新專案

git init

就會開好新專案

你現在要從網路上 (remote) 把專案複製下來

git clone [ssh or https path]

就會把專案複製下來

你要為專案設定新的遠端倉庫 (remote)

git remote add [remote_name] [ssh or https path]

就可以開始跟 remote 互動

你要為遠端倉庫 (remote) 修改路徑

git remote set-url [remote_name] [ssh or https path]

就可以改 remote 的路徑

你要看遠端倉庫 (remote) 有哪些

git remote -v

要查看現在狀態

git status

就可以看到跟 HEAD 的差距

要查看現在跟 HEAD 的檔案修改內容

git diff HEAD

就可以看到跟 HEAD 之間修改哪幾行

要查看之前的 commit 有哪些

git log

要把檔案加到下一次 commit 的內容

git add [file_path]

就把檔案加到 staging area

可以用 git status 看效果

要把所有有改變的檔案加到下一次 commit 的內容

git add --all

就把所有變更檔案加到 staging area 的狀態存檔

可以用 git status 看效果

要把 staging area 的內容產生一個紀錄點

git commit -m ["my_commit message"]

就可以產生一個以後可以跳回來的記錄點！

要把 commit 的內容同步到遠端 (remote)

git push [remote_name] [branch_name]

預設的 remote_name 通常是 origin, 預設的 branch_name 通常是 master

要把 commit 的內容從遠端 (remote) 同步到自己電腦 (local)

git pull [remote_name] [branch_name]

預設的 remote_name 通常是 origin, 預設的 branch_name 通常是 master

要看現在有哪些分支

git branch

要開一個新的分支

git branch [branch_name]

要移動到另一個分支

git checkout [branch_name]

要把另一個分支的內容合併進來

git merge [branch_name]
